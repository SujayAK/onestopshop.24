2. OPTION 1: SUPABASE (RECOMMENDED)
================================================================================

What is Supabase?
- PostgreSQL database manageable through browser UI
- Built-in authentication (signup/login)
- Real-time database
- File storage (for product images/videos)
- Free tier: 500MB storage, up to 5 concurrent connections
- UI is beginner-friendly - no coding needed

SETUP STEPS (5 minutes):

1. Go to https://supabase.com and click "Start your project"
2. Sign up with Email/GitHub
3. Click "New project"
4. Name it: "onestopshop"
5. CreatePassword for database
6. Click "Create new project" (wait 2-3 min)

YOU NOW HAVE A BACKEND!

CREATING TABLES (Visual, no SQL):

1. Go to "Table Editor" on left sidebar
2. Click "Create new table" → "New table"
3. Create these table:

TABLE 1: users
- id (auto)
- email (text, unique)
- password (text)
- fname (text)
- lname (text)
- phone (text)
- address (text)
- city (text)
- postal_code (text)
- avatar_url (text)
- created_at (timestamp, auto)

TABLE 2: products
- id (auto)
- name (text)
- description (text)
- category (text)
- price (number)
- stock_quantity (number)
- image_url (text)
- created_at (timestamp, auto)

TABLE 3: orders
- id (auto)
- user_id (number, link to users)
- order_number (text, unique)
- total_amount (number)
- status (text): "pending", "confirmed", "shipped"
- payment_status (text): "pending", "completed"
- razorpay_order_id (text)
- razorpay_payment_id (text)
- shipping_address (text)
- created_at (timestamp, auto)

TABLE 4: coupons
- id (auto)
- code (text, unique)
- discount_type (text): "percentage" or "fixed"
- discount_value (number)
- start_date (timestamp)
- end_date (timestamp)
- max_uses (number)
- min_cart_value (number)

MANAGING DATA (All in Browser):

1. Go to "Table Editor"
2. Click on any table to see all data
3. Click "+" to add new row
4. Edit any cell directly
5. Delete rows by clicking "..." menu

Upload Product Images:
1. Go to "Storage" on left sidebar
2. Click "Create new bucket" → name it "products"
3. Enable public access
4. Drag & drop images to upload
5. Copy public URL and paste in product's image_url field

AUTHENTICATION (Login/Signup):
- Supabase handles all authentication automatically
- Your frontend login form connects directly
- No need to manage passwords manually

DATABASE URL & API KEY:
1. Go to "Project Settings" → "Database"
2. Connection string: postgresql://user:password@host:5432/database
3. Go to "API" or "Settings" → copy your "Anon Public Key"
4. Your frontend uses this to connect

3. OPTION 2: FIREBASE (GOOGLE'S SOLUTION)
================================================================================

What is Firebase?
- Google's backend platform
- Very beginner-friendly
- Real-time database (NoSQL)
- Automatic hosting
- Free tier: 1GB storage, unlimited reads

SETUP STEPS (3 minutes):

1. Go to https://firebase.google.com
2. Click "Get Started"
3. Sign in with Google account
4. Click "Create a project"
5. Name it "onestopshop"
6. Click "Continue" → "Create project"

CREATING COLLECTIONS (Visual):

1. Go to "Firestore Database" on left
2. Click "Create database"
3. Select "Start in test mode" (for development)
4. Choose region closest to you
5. Click "Create"

CREATE COLLECTIONS:

1. Click "+ Add collection" → name it "users"
2. Click "Auto ID" → "Save"
3. Add fields:
   - email (string)
   - password (string)
   - fname (string)
   - lname (string)
   - phone (string)
   - created_at (timestamp)

4. Repeat for "products", "orders", "coupons"

ADDING DATA:

1. Click on collection name (e.g., "users")
2. Click "Add document"
3. Click "Auto ID" to generate unique ID
4. Add each field
5. Click "Save"

FILE UPLOAD (Images):

1. Go to "Storage" on left
2. Click "Get started"
3. Click "Create a bucket"
4. Name it "product-images"
5. Choose rules "Start in test mode"
6. Drag & drop images to upload

VIEW & MANAGE DATA:

- Real-time updates: Changes appear instantly
- Click any document to edit
- Delete documents with "..." menu

3. Click "+" to add item

BEAUTIFUL ADMIN INTERFACE:

- Drag to reorder products

CONNECT TO YOUR FRONTEND:

1. Go to Account settings → API

AUTOMATE TASKS:

7. STEP-BY-STEP SETUP GUIDE (USING SUPABASE - RECOMMENDED)

TOTAL TIME: 15 minutes to working backend

STEP 1: Create Supabase Project
────────────────────────────────
1. Visit https://supabase.com
2. Click "Start your project"
3. Sign up with email or GitHub
4. Click "New Project"
5. Fill:
   - Name: onestopshop
   - Database Password: ChooseStrongPassword123!
   - Region: Closest to your users
6. Click "Create new project"
7. Wait 2-3 minutes for setup

STEP 2: Create Database Tables
──────────────────────────────
After project loads, you see "Table Editor" on left. Click it.

A. Users Table:
   1. Click "Create table" → "New table"
   2. Name: users
   3. Add columns (click "+"):
      - id (auto-selected as Primary Key)
      - email (Type: char, Unique: YES)
      - password (Type: char)
      - fname (Type: char)
      - lname (Type: char)
      - phone (Type: char)
      - address (Type: text)
      - city (Type: char)
      - avatar_url (Type: char)
      - created_at (Type: timestamp, Default: now())
   4. Click "Save"

B. Products Table:
   1. Create table → name: products
   2. Add columns:
      - id (auto)
      - name (char)
      - description (text)
      - category (char)
      - price (Type: numeric)
      - stock_quantity (numeric)
      - image_url (char)
      - created_at (timestamp, default: now())
   3. Save

C. Orders Table:
   1. Create table → name: orders
   2. Columns:
      - id (auto)
      - user_id (Type: integer, Foreign Key to users.id)
      - order_number (char, Unique)
      - total_amount (numeric)
      - tax_amount (numeric)
      - status (char, Default: 'pending')
      - payment_status (char, Default: 'pending')
      - razorpay_order_id (char)
      - razorpay_payment_id (char)
      - created_at (timestamp, default: now())
   3. Save

D. Coupons Table:
   1. Create table → name: coupons
   2. Columns:
      - id (auto)
      - code (char, Unique)
      - discount_type (char)
      - discount_value (numeric)
      - start_date (timestamp)
      - end_date (timestamp)
      - max_uses (integer)
      - min_cart_value (numeric)
   3. Save

STEP 3: Enable Authentication
──────────────────────────────
1. Go to "Authentication" on left sidebar
2. Click "Providers" under Authentication menu
3. Email/Password is enabled by default
4. Go back to "Users" tab
5. Enable "Email confirmations" if you want verification emails

STEP 4: Set Up File Storage
───────────────────────────
1. Go to "Storage" on left sidebar
2. Click "Create new bucket"
3. Name: products
4. Uncheck "Private bucket" to make images public
5. Click "Create bucket"
6. Done! Drag & drop images to upload

STEP 5: Get Your Connection Credentials
────────────────────────────────────────
1. Go to "Settings" (gear icon) → "Project"
2. Copy "Project URL": example: https://xxxxx.supabase.co
3. Go to "API" tab in Settings

8. CONNECTING FRONTEND TO BACKEND
================================================================================

Your frontend (onestopshop) needs to connect to Supabase API.

Install Supabase library:
```
npm install @supabase/supabase-js
```

Create new file: src/utils/supabaseClient.js
```javascript
import { createClient } from '@supabase/supabase-js'

const supabaseUrl = 'https://xxxxx.supabase.co' // From Supabase settings
const supabaseAnonKey = 'eyJhbGc...' // From Supabase API tab

export const supabase = createClient(supabaseUrl, supabaseAnonKey)
```

Example: Fetch products:
```javascript
import { supabase } from './utils/supabaseClient'

// In your code:
const { data: products, error } = await supabase
  .from('products')
  .select('*')
  .eq('category', 'Bags')

console.log(products) // Array of products
```

Example: Add order:
```javascript
const { data, error } = await supabase
  .from('orders')
  .insert([
    {
      user_id: userId,
      order_number: 'ORD-12345',
      total_amount: 5000,
      status: 'pending'
    }
  ])
```

Example: User signup:
```javascript
const { data, error } = await supabase.auth.signUp({
  email: 'user@example.com',
  password: 'password123'
})
```

Everything is REST API calls. No backend server needed!
